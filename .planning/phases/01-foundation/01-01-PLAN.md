  <action>
    1. Run `npx create-next-app@15 . --typescript --tailwind --eslint --app --no-src-dir --import-alias "@/*"` to scaffold into the current directory. CRITICAL: Use `@15` not `@latest` -- `@latest` installs Next.js 16.2.1 which is the wrong major version.

    2. After scaffold completes, verify `package.json` shows `"next": "15.x.x"` (not 16.x). If it shows 16.x, stop and fix.

    3. Upgrade Tailwind to v4 with the correct PostCSS plugin. Run:
       `npm install tailwindcss@latest @tailwindcss/postcss postcss`

    4. Replace `postcss.config.mjs` content with the v4 pattern (per RESEARCH.md Pattern 1):
       ```javascript
       const config = {
         plugins: {
           "@tailwindcss/postcss": {},
         },
       };
       export default config;
       ```
       CRITICAL: Do NOT use the v3 pattern `{ tailwindcss: {} }` -- it silently produces no styles with v4.

    5. Replace `app/globals.css` with Tailwind v4 CSS-first config:
       ```css
       @import "tailwindcss";

       @theme {
         --font-display: var(--font-playfair-display);
         --font-body: var(--font-inter);
       }
       ```
       Remove any Tailwind v3 `@tailwind base/components/utilities` directives if present. Remove any `tailwind.config.ts` or `tailwind.config.js` file that the scaffold created -- v4 uses CSS-first config only.

    6. Update `app/layout.tsx` with font loading and metadataBase (per RESEARCH.md Patterns 2 and 3):
       - Import `Playfair_Display` and `Inter` from `next/font/google`
       - Configure both with `subsets: ['latin']`, `display: 'swap'`, and CSS variable names `--font-playfair-display` and `--font-inter`
       - Set `metadataBase` using `process.env.VERCEL_PROJECT_PRODUCTION_URL` with localhost:3000 fallback
       - Set `title: { default: 'DJ Catan — Developer', template: '%s | DJ Catan' }`
       - Set `description: 'Full-stack developer building thoughtful digital products.'`
       - Apply font variables to `<html>` className: `${playfairDisplay.variable} ${inter.variable}`
       - Apply `font-body antialiased` to `<body>` className
       - Set `lang="en"` on `<html>`

    7. Replace `app/page.tsx` with a minimal placeholder that confirms fonts and Tailwind work:
       ```tsx
       export default function Home() {
         return (
           <main className="flex min-h-screen flex-col items-center justify-center p-8">
             <h1 className="font-display text-5xl font-bold">DJ Catan</h1>
             <p className="mt-4 text-lg text-gray-600">
               Portfolio coming soon.
             </p>
           </main>
         )
       }
       ```

    8. Run `npm run build` to verify the scaffold compiles without errors.
  </action>
