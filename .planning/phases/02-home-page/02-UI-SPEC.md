## Spacing Scale

Declared values (multiples of 4 — 8-point scale):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1` / `p-1` | Icon gaps, inline element padding |
| sm | 8px | `gap-2` / `p-2` | Compact element spacing, badge inner padding |
| md | 16px | `gap-4` / `p-4` | Default element spacing, card inner padding |
| lg | 24px | `gap-6` / `px-6` | Section horizontal padding, card section gaps |
| xl | 32px | `gap-8` / `p-8` | Layout column gaps |
| 2xl | 48px | `py-12` | Minor section vertical padding |
| 3xl | 64px | `py-16` | Major section vertical padding (About, Projects, Resume) |
| hero | 96px | `py-24` | Hero section vertical breathing room |

Exceptions:
- Navbar height: 64px (`h-16`) — fixed, not from scale
- Touch targets (hamburger button, icon links): minimum 44px hit area via padding even if icon is smaller
- Photo placeholder max-width: `max-w-xs` (320px) — not from spacing scale, from layout constraint
- Scroll chevron: `bottom-8` (32px) from bottom of hero

Source: RESEARCH.md Architecture Patterns (Pattern 2, Pattern 6, Pattern 7) + CONTEXT.md Claude's Discretion

### 60 / 30 / 10 Split

| Role | Hex | HSL | Usage |
|------|-----|-----|-------|
| Dominant (60%) | `#FAF8F5` | `36 33% 97%` | Page background, navbar bg, all section backgrounds, footer bg |
| Secondary (30%) | `#FFFFFF` | `0 0% 100%` | Project cards, photo placeholder container — provides subtle lift over cream |
| Accent (10%) | `#C2603A` | `18 55% 49%` | Primary CTA button ("View My Work"), download button ("Download Resume"), footer social icon hover state, navbar link active/hover state |

Accent reserved for exactly:
1. "View My Work" CTA button (Hero) — filled button
2. "Download Resume" download button (Resume section) — filled button
3. Footer social icon links on hover — color change from `--muted-foreground` to `--primary`
4. Navbar links on hover — color change from `--foreground` to `--primary`
5. Focus ring on all interactive elements (`--ring`)

Accent is NOT used for: section headings, body text, card borders, badge backgrounds, tech stack pills.

## Architecture Notes (for executor reference)

| Decision | Rationale |
|----------|-----------|
| CSS variables format | Preserve existing space-separated HSL format (no `hsl()` wrapper in var definition). Double-wrapping `hsl(hsl(...))` breaks all color utilities. |
| `scroll-smooth` on `<html>` | Must also add `data-scroll-behavior="smooth"` attribute — Next.js 15 requires this to coordinate with its scroll restoration |
| Navbar z-index | `z-50` on header only. Hero section: `relative` with no z-index. Prevents stacking context conflicts. |
| Server vs Client | All sections (Hero, About, Projects, Resume, Footer) are Server Components. Only `NavbarMobile` (hamburger toggle) requires `'use client'`. |
| Plain `<a>` for anchors | Use `<a href="#section-id">` for in-page anchor scrolling, not `<Link>` — avoids Next.js router overhead for same-page navigation |
| `public/resume.pdf` | Must exist before the Resume component ships. A 1KB blank PDF is sufficient as placeholder. |
