  <name>Task 2: Create Zod schema, Server Action, and env configuration</name>
  <files>lib/schemas/contact.ts, app/actions/contact.ts, .env.local, .env.example</files>
  <action>
1. Create `lib/schemas/contact.ts` — shared Zod schema (per D-04, D-05):
   - `contactSchema`: z.object with name (string, min 1, "Name is required"), email (string, email, "Please enter a valid email address"), message (string, min 10, "Message must be at least 10 characters")
   - Export `ContactFormValues` type via `z.infer<typeof contactSchema>`
   - This schema is imported by BOTH the Server Action (server-side re-validation) and ContactForm.tsx (client-side validation via zodResolver) in Plan 02

2. Create `app/actions/contact.ts` — Server Action (per D-12, D-13, D-14, D-15, D-18):
   - `'use server'` directive at top
   - Import `Resend` from 'resend' and `contactSchema` from '@/lib/schemas/contact'
   - Define `ActionResult` type: `{ success: boolean; error?: string }`
   - Export `sendContactEmail` function with signature: `(prevState: ActionResult, formData: FormData) => Promise<ActionResult>`
   - Implementation order inside the function:
     a. Honeypot check (per D-06, CONTACT-04): `const honeypot = formData.get('website')` — if truthy, return `{ success: true }` silently (do NOT send email)
     b. Env var guard (per D-13): if `!process.env.RESEND_API_KEY`, return `{ success: false, error: 'Configuration error' }`
     c. Parse FormData through `contactSchema.safeParse()` — if invalid, return `{ success: false, error: 'Please check your form fields' }`
     d. Instantiate `new Resend(process.env.RESEND_API_KEY)` INSIDE the function body (NOT module scope — build would fail, see Research pitfall 2)
     e. Call `resend.emails.send()` with:
        - from: `'Portfolio Contact <onboarding@resend.dev>'` (sandbox domain, per D-14)
        - to: `['hello@djcatan.com']` (per D-14, placeholder)
        - replyTo: visitor's email from parsed data
        - subject: `New portfolio contact from ${name}` (per D-15)
        - html: simple HTML string with name, email, message (per D-15). Escape newlines in message with `.replace(/\n/g, '<br>')`
     f. If Resend returns error, return `{ success: false, error: 'Something went wrong. Please try emailing directly.' }`
     g. On success, return `{ success: true }`

3. Create `.env.local` with `RESEND_API_KEY=` (empty value — user fills in their key). This file is already gitignored by Next.js default .gitignore.

4. Create `.env.example` with `RESEND_API_KEY=your_resend_api_key_here` and a comment explaining where to get the key. This file IS committed to git.
