**File 1: `components/sections/ContactForm.tsx`** (per D-17, D-19 — `'use client'` component)

Create the form client component with these specifics:

1. **Imports:**
   - `'use client'` directive at top
   - `useRef`, `useState`, `useTransition` from 'react'
   - `useActionState` from 'react'
   - `useForm` from 'react-hook-form'
   - `zodResolver` from '@hookform/resolvers/zod'
   - `contactSchema`, `ContactFormValues` from '@/lib/schemas/contact'
   - `sendContactEmail` from '@/app/actions/contact'
   - `Input` from '@/components/ui/input'
   - `Textarea` from '@/components/ui/textarea'
   - `Label` from '@/components/ui/label'
   - `Button` from '@/components/ui/button'

2. **State setup:**
   - `const formRef = useRef<HTMLFormElement>(null)`
   - `const [isPending, startTransition] = useTransition()`
   - `const [state, formAction] = useActionState(sendContactEmail, { success: false })`
   - `const [showSuccess, setShowSuccess] = useState(false)` — local flag for "Send another" reset (see Research pitfall 3)
   - `useForm<ContactFormValues>` with `resolver: zodResolver(contactSchema)` and `mode: 'onBlur'` (per Research pitfall 6)
   - Effect or check: when `state.success` becomes true, set `showSuccess(true)`

3. **Submit handler (per D-19, Research pattern 3):**
   ```typescript
   const onSubmit = handleSubmit(() => {
     startTransition(() => {
       formAction(new FormData(formRef.current!))
     })
   })
   ```
   CRITICAL: Must use `startTransition` — without it, React 19 throws "async function dispatched outside of action context" (Research pitfall 1).

4. **Success state (per D-09, D-22, D-23):**
   - When `showSuccess` is true, render a confirmation block instead of the form:
     - Heading: "Message sent!" (`font-display text-2xl font-bold text-foreground`)
     - Body: "Thanks -- I'll get back to you soon." (`text-muted-foreground mt-2`)
     - "Send another" button/link that calls `reset()` (RHF), sets `showSuccess(false)`, and resets `state` by re-calling initial state

5. **Error state (per D-10, D-24):**
   - If `state.error` exists and `!showSuccess`, show error message ABOVE the form:
     - "Something went wrong. Please try emailing directly at hello@djcatan.com" (per D-24, D-25)
     - The email is a `mailto:` link styled as `text-primary underline`
     - Container: `rounded-md bg-destructive/10 p-4 text-sm text-destructive`

6. **Form fields (per D-04, D-07, D-08):**
   - Each field wrapped in `<div className="space-y-2">`:
     a. **Name:** `<Label htmlFor="name">Your name</Label>` + `<Input id="name" placeholder="e.g. Alex Chen" {...register('name')} />` + error message
     b. **Email:** `<Label htmlFor="email">Your email</Label>` + `<Input id="email" type="email" placeholder="e.g. alex@example.com" {...register('email')} />` + error message
     c. **Message:** `<Label htmlFor="message">Your message</Label>` + `<Textarea id="message" placeholder="Tell me about your project or just say hi..." className="min-h-[120px]" {...register('message')} />` + error message
   - Inline errors: `{errors.fieldName && <p id="fieldName-error" className="text-sm text-destructive">{errors.fieldName.message}</p>}`
   - Add `aria-describedby={errors.fieldName ? 'fieldName-error' : undefined}` to each input for accessibility
   - Fields container: `<div className="space-y-6">`

7. **Honeypot field (per D-06, CONTACT-04, Research pattern 4):**
   ```tsx
   <input
     {...register('website' as any)}
     name="website"
     type="text"
     tabIndex={-1}
     autoComplete="off"
     aria-hidden="true"
     className="sr-only"
   />
   ```
   Place after the visible fields, before the submit button. The `as any` cast is needed because 'website' is not in the Zod schema type.

8. **Submit button (per D-11, D-22):**
   - `<Button type="submit" disabled={isPending} className="w-full">`
   - Text: `isPending ? 'Sending...' : 'Send Message'`

9. **Form element:**
   - `<form ref={formRef} onSubmit={onSubmit} className="space-y-6" noValidate>`
   - `noValidate` disables browser default validation (RHF handles it)
